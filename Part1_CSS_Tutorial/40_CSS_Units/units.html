There are two types of length units: absolute and relative.


1. Absolute

Absolute length units are not recommended for use on screen

1. cm
2. mm
3. in, 1in = 2.54cm
px , 1px = 1/96 inch
pt , 1pt = 1/72 inch
pc , 1pc = 12pt

* Pixels (px) are relative to the viewing device. For low-dpi devices, 1px is one device pixel (dot) of the display. For printers and high resolution screens 1px implies multiple device pixels.


2. Relative

1. em : parent font-size times . i.e 2em means 2 times parent font-size. (can cause compounding issue, during nesting.) { if not define then took the parent font size. }

note: if you set the font size for the current element it wont consider the parent and use yours define font size for em.

2. ex : 1ex = height of the small letters like a, b, c (inconsistent since , every devices font size may vary.) { current element font size only}

3. ch : 1ch equals the width of the "0" character in the current font.

4. rem : Relative to font-size of the root element. (solves the compounding issue in the em during nesting.)

5. vw : 1vw = 1% of the viewport width.

6. vh : 1vh = 1% of the viewport height.

7. vmax : 1vmax = 1% of the largest dimension of the viewpport.

8. vmin : 1vmin = 1% of the smallest dimension of the viewport.

9. % : relative to the parent element.

* Viewport = the browser window size. If the viewport is 50cm wide, 1vw = 0.5cm.
